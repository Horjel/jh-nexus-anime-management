📘 MEMORIA DEL PROYECTO
Sistema de Gestión de Tienda “Otaku Center”
1. Introducción

El presente documento describe el análisis, diseño y desarrollo de una aplicación web denominada Otaku Center, orientada a la gestión integral de una tienda de productos relacionados con el mundo del anime y manga.

El sistema permitirá administrar productos, categorías, clientes y pedidos, aplicando una arquitectura en capas basada en el patrón MVC (Modelo-Vista-Controlador) y el uso de tecnologías modernas del ecosistema Java.

El objetivo principal del proyecto es consolidar conocimientos en desarrollo backend con Spring Boot, persistencia con JPA/Hibernate y estructuración profesional de aplicaciones empresariales.

2. Objetivos del Proyecto
2.1 Objetivo General

Desarrollar una aplicación web robusta y escalable para la gestión de una tienda otaku, aplicando buenas prácticas de ingeniería del software.

2.2 Objetivos Específicos
Implementar una arquitectura por capas (JSP, Controller, Service, DAO).
Gestionar operaciones CRUD sobre las entidades principales.
Integrar persistencia mediante JPA y base de datos relacional.
Aplicar validaciones y control de lógica de negocio.
Desarrollar una interfaz web funcional mediante JSP.
Documentar el sistema para su publicación en GitHub.
3. Alcance del Sistema

El sistema permitirá:

Gestión de productos (altas, bajas, modificaciones y consultas).

5. Arquitectura del Sistema

El sistema sigue una arquitectura en capas basada en el patrón MVC:

5.1 Vista (JSP)

Encargada de la presentación de datos al usuario.

Funciones:

Formularios de entrada
Listados de datos
Navegación entre vistas
5.2 Controlador (Controller)

Gestiona las peticiones HTTP.

Responsabilidades:

Recibir solicitudes del usuario
Invocar servicios
Devolver vistas
5.3 Servicio (Service)

Contiene la lógica de negocio.

Responsabilidades:

Validaciones
Reglas del sistema
Coordinación entre capas
5.4 Acceso a Datos (DAO)

Gestiona la interacción con la base de datos.

Responsabilidades:

Operaciones CRUD
Uso de JPA / EntityManager
5.5 Modelo (Entity)

Representa las entidades del sistema.

Ejemplo: Registro de Producto

El Controller invoca el Service
El Service valida datos
El Service llama al DAO
El DAO guarda en la base de datos
Se devuelve la respuesta a la vista
